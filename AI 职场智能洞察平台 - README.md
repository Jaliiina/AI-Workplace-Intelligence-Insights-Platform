# 1. 项目概述

## 1.1 项目背景

随着人工智能技术的快速发展，AI 领域人才需求激增，招聘市场呈现出岗位类型多样、地域分布不均、技能要求差异化大等特点。企业招聘方难以快速掌握市场全貌，求职者也面临职业方向迷茫、技能提升无头绪等问题。基于此，AI 职场智能洞察平台应运而生，旨在通过数据采集、清洗、分析与可视化，为用户提供全方位的 AI 职场数据洞察服务。

## 1.2 核心价值

- 为求职者：提供岗位趋势、薪资水平、技能要求等数据，助力职业规划与技能提升方向决策；

- 为招聘方：呈现市场人才供给、区域竞争格局、技能匹配度等信息，优化招聘策略与岗位定价；

- 为研究者：提供 AI 行业职场发展的客观数据支撑，辅助行业研究与趋势预判。

## 1.3 适用人群

AI 相关专业求职者、企业 HR 与招聘负责人、AI 行业研究者、职业规划咨询师、高校就业指导老师等。

# 2. 核心功能模块详解

## 2.1 主大屏 · AI 市场全景

作为平台的核心入口，主大屏以全景式视角呈现 AI 招聘市场的核心数据，所有图表支持实时交互（缩放、悬停查看详情、筛选）。核心展示内容包括：

- 时间趋势分析：近 12 个月 AI 岗位数量与平均薪资的月度变化曲线，支持按岗位类别筛选查看；

- 岗位结构玫瑰图：多维度展示不同岗位类别的占比分布（如算法工程师、数据工程师、AI 产品经理等）；

- 全国 Geo 气泡图：以气泡大小表示岗位数量，颜色表示平均薪资，直观呈现全国各省份 AI 岗位分布格局；

- 城市排名榜单：按岗位数量、平均薪资、增长速度三个维度展示 Top10 城市，支持切换排序维度；

- 学历漏斗分析：展示不同学历层次（大专、本科、硕士、博士）在 AI 岗位中的需求占比，呈现学历门槛分布；

- 热门技能 Top10：展示当前 AI 市场需求最高的 10 项技能，包含技能名称、需求占比及对应平均薪资。

## 2.2 岗位结构大屏

聚焦 AI 岗位的内部结构拆解，通过多维度交叉分析，帮助用户深入了解不同类型岗位的需求差异。核心功能包括：

- 多维度交叉分析：支持按「岗位类别 × 学历要求」「岗位类别 × 工作经验」「岗位类别 × 区域分布」「岗位类别 × 技能要求」进行交叉筛选，查看对应维度的岗位数量、薪资分布数据；

- 岗位赛道对比：对比不同 AI 赛道（如计算机视觉、自然语言处理、大模型应用、数据分析等）的岗位数量、平均薪资、技能需求差异；

- 岗位要求详情：点击具体岗位类别，可查看该类岗位的核心职责、必备技能、优先技能、薪资范围等详细信息；

- 趋势对比：支持查看近 6 个月不同岗位类别的数量变化趋势，预判岗位需求热度走向。

## 2.3 城市与区域分析模块

专注于地域维度的 AI 职场数据洞察，为用户提供城市级别的精细化分析。核心功能包括：

- 城市数据总览：选择具体城市后，展示该城市 AI 岗位总数、平均薪资、岗位类别分布、学历要求分布、经验要求分布等核心数据；

- 区域热力图：展示目标城市内不同行政区的岗位分布热力，直观呈现岗位聚集区域；

- 城市间对比：支持选择 2-4 个城市进行多维度对比，包括岗位数量、平均薪资、技能需求、学历门槛等；

- 城市岗位趋势：展示目标城市近 12 个月 AI 岗位数量与薪资的变化趋势，分析城市市场波动；

- 区域特色岗位分析：呈现不同城市的特色 AI 岗位（如北京的大模型相关岗位、深圳的计算机视觉岗位等）及对应技能需求。

## 2.4 技能图谱与人才画像模块

围绕 AI 领域技能与人才核心要素，构建可视化图谱，助力用户明确技能提升方向与人才匹配标准。核心功能包括：

- 技能词云展示：以词云形式直观呈现当前 AI 市场的热门技能，词的大小对应技能需求频率；

- 技能关联图谱：展示技能之间的关联关系（如 Python 技能常与机器学习、数据分析技能关联），帮助用户理解技能组合逻辑；

- 技能-薪资关联分析：展示不同技能对应的平均薪资水平、岗位数量，为技能提升提供价值参考；

- 标准化人才画像：展示不同岗位类别的典型人才画像，包括学历、工作经验、核心技能、附加技能、项目经验要求等；

- 技能差距分析：支持用户输入自身技能清单，系统对比目标岗位的技能要求，生成技能差距报告。

## 2.5 数据导出与分享功能

支持用户将平台的分析数据与图表进行导出和分享，提升数据利用效率。核心功能包括：

- 数据导出：支持将当前页面的表格数据导出为 Excel、CSV 格式，图表导出为 PNG、SVG 格式；

- 报告生成：支持自动生成基于当前筛选条件的数据分析报告（Word/PDF 格式），包含数据图表与核心结论；

- 链接分享：支持生成当前页面的临时分享链接，他人可通过链接查看对应的数据可视化页面（有效期可设置）；

- 嵌入支持：提供 iframe 嵌入代码，支持将指定数据图表嵌入到第三方网站或内部系统。

# 3. 数据体系说明

## 3.1 数据来源

平台数据来源于多渠道的合法合规采集与整合，主要包括：

- 主流招聘平台公开数据：如 Boss 直聘、智联招聘、前程无忧等平台的 AI 相关岗位公开信息；

- 企业官方招聘渠道数据：各科技企业官网、官方招聘公众号发布的 AI 岗位信息；

- 行业研究报告数据：整合艾瑞咨询、易观分析、IDC 等机构发布的 AI 行业人才相关报告数据；

- 高校就业数据：部分高校发布的 AI 相关专业毕业生就业去向与薪资数据（已脱敏处理）；

- 用户贡献数据：经用户授权的匿名化职场信息（如薪资水平、技能匹配度等），用于优化数据模型。

## 3.2 数据维度

平台数据涵盖 6 大核心维度，形成完整的 AI 职场数据体系：

- 岗位基础维度：岗位名称、岗位类别、所属行业、企业类型、企业规模、工作地点；

- 薪资维度：平均薪资、薪资范围、薪资结构（基本工资、绩效、奖金）、薪资与经验/学历的关联数据；

- 要求维度：学历要求、工作经验要求、技能要求（核心技能、优先技能）、证书要求；

- 时间维度：岗位发布时间、数据统计周期（月度/季度/年度）、岗位需求变化趋势；

- 地域维度：省份、城市、行政区、产业园区等不同层级的地域分布数据；

- 企业维度：企业所属领域、企业招聘热度、企业平均薪资水平、企业技能需求偏好。

## 3.3 数据更新机制

为保证数据的时效性与准确性，平台采用「定期全量更新 + 实时增量补充」的更新机制：

- 全量更新：每月 1 日对全量数据进行采集与更新，覆盖所有核心数据维度；

- 增量补充：每日凌晨对前 24 小时内新增的岗位信息进行增量采集，确保最新岗位数据及时上线；

- 数据校验：每次更新后，通过机器校验 + 人工抽样审核的方式，对数据的准确性、完整性进行校验，剔除无效数据（如重复岗位、虚假薪资、无效联系方式等）；

- 更新日志：平台后台记录详细的数据更新日志，包含更新时间、更新数据量、更新维度等信息，便于追溯与问题排查。

## 3.4 数据质量说明

平台通过多重机制保障数据质量，但仍存在部分局限性：

- 数据准确性：通过数据交叉验证、重复数据去重、无效数据过滤等机制提升准确性，人工抽样审核准确率≥95%；

- 数据完整性：覆盖全国 34 个省级行政区、100+ 重点城市的 AI 相关岗位，核心岗位类别的数据覆盖率≥90%；

- 局限性说明：部分中小企业的岗位信息可能未被完全采集，薪资数据为招聘方标注的范围，实际薪资可能受个人能力、谈判技巧等因素影响，数据仅供参考。

# 4. 技术架构详解

## 4.1 整体架构图

平台采用前后端分离的微服务架构，整体分为数据采集层、数据处理层、服务层、前端展示层四个核心层级，架构如下：

```text

┌───────────────────┐
│  前端展示层       │  HTML/CSS/JS + ECharts + Vue3
└────────┬──────────┘
         │
┌────────▼──────────┐
│  服务层           │  接口服务、权限服务、数据导出服务、分享服务
└────────┬──────────┘
         │
┌────────▼──────────┐
│  数据处理层       │  数据清洗、数据转换、数据建模、数据分析
└────────┬──────────┘
         │
┌────────▼──────────┐
│  数据采集层       │  爬虫采集、API 对接、数据录入
└────────┬──────────┘
         │
┌────────▼──────────┐
│  数据存储层       │  MySQL + Redis + MinIO
└───────────────────┘
```

## 4.2 前端技术栈

- 核心框架：Vue3（Composition API 语法），实现组件化开发与状态管理；

- 路由管理：Vue Router，实现页面路由跳转与权限控制；

- 状态管理：Pinia，管理全局状态，优化组件间数据通信；

- UI 组件库：Element Plus，提供统一的 UI 风格与基础组件；

- 可视化库：ECharts 5.x，实现各类图表（折线图、柱状图、饼图、地图、词云、漏斗图等）的渲染与交互；

- 样式解决方案：Scss + CSS Modules，实现样式模块化与复用；

- 数据请求：Axios，处理前后端数据交互，实现请求拦截、响应拦截、错误处理；

- 其他工具：Vite（构建工具，提升开发与构建效率）、ESLint（代码规范检查）、Prettier（代码格式化）。

## 4.3 后端技术栈

- 核心框架：Python + FastAPI，实现高性能的 API 接口开发，支持异步请求处理；

- 数据采集：Scrapy（爬虫框架，用于采集招聘平台公开数据）、Requests（API 对接与数据请求）；

- 数据处理：Pandas（数据清洗与转换）、NumPy（数值计算）、Scikit-learn（简单的数据分析与建模）；

- 权限认证：JWT（JSON Web Token），实现用户身份认证与权限控制；

- 任务调度：Celery + Redis，实现定时数据采集、数据更新等异步任务调度；

- 日志处理：Loguru，实现高效的日志记录与管理；

- API 文档：FastAPI 自带的 Swagger UI，自动生成 API 文档，便于接口调试。

## 4.4 数据存储方案

- 关系型数据库：MySQL 8.0，用于存储结构化数据，如用户信息、岗位基础信息、权限数据等；

- 缓存数据库：Redis 6.x，用于缓存热点数据（如热门技能、城市排名等）、存储用户会话信息、支持 Celery 任务调度；

- 文件存储：MinIO，用于存储用户导出的报告文件、图表图片、界面截图等静态资源；

- 数据备份：采用 MySQL 主从复制实现数据备份，每日自动备份全量数据，保留近 30 天的备份记录。

## 4.5 可视化技术实现

平台的可视化功能基于 ECharts 5.x 实现，针对不同类型的图表采用针对性的实现方案：

- 基础图表（折线图、柱状图、饼图）：直接使用 ECharts 基础组件，通过配置项自定义样式与交互逻辑；

- 地图类图表（Geo 气泡图、热力图）：基于 ECharts 地图组件，整合全国/城市级 GeoJSON 数据，实现地域数据的可视化；

- 词云图：使用 ECharts 词云扩展组件，通过调整词的大小、颜色、旋转角度等参数，提升视觉效果；

- 关联图谱：基于 ECharts 关系图组件，通过节点与边的配置，展示技能之间的关联关系；

- 交互优化：实现图表的hover提示（展示详细数据）、点击筛选（下钻查看细分数据）、缩放（适配不同屏幕）、图例切换（显示/隐藏指定数据系列）等交互功能；

- 响应式适配：通过监听窗口大小变化，动态调整图表的尺寸与布局，确保在不同设备（PC、平板、手机）上均能正常展示。

## 5.2 本地开发环境搭建步骤

### 步骤 1：克隆项目代码

```bash

# 克隆代码仓库
git clone https://github.com/xxx/ai-workplace-insight-platform.git
# 进入项目根目录
cd ai-workplace-insight-platform
```

### 步骤 2：搭建后端开发环境

```bash

# 进入后端目录
cd backend
# 创建虚拟环境
python -m venv venv
# 激活虚拟环境（Windows）
venv\Scripts\activate
# 激活虚拟环境（Mac/Linux）
source venv/bin/activate
# 安装后端依赖
pip install -r requirements.txt
# 初始化数据库（创建数据表、导入基础数据）
python manage.py init_db
# 启动后端开发服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

后端服务启动后，可访问 `http://localhost:8000/docs` 查看 API 文档。

### 步骤 3：搭建前端开发环境

```bash

# 进入前端目录（新终端窗口）
cd frontend
# 安装前端依赖
npm install
# 启动前端开发服务
npm run dev
```

前端服务启动后，可访问 `http://localhost:5173` 进入平台页面。

### 步骤 4：配置数据存储服务

1. 启动 MinIO 服务（参考 MinIO 官方文档：[https://min.io/docs/minio/linux/operations/installation.html](https://min.io/docs/minio/linux/operations/installation.html)）；

2. 在 MinIO 中创建存储桶（如 `ai-workplace-data`）；

3. 修改后端配置文件 `backend/config/config.yaml` 中的 MinIO 相关配置（地址、端口、访问密钥、存储桶名称）。

### 步骤 5：启动数据采集服务（可选）

```bash

# 进入爬虫目录
cd backend/spider
# 启动爬虫采集数据（全量采集）
scrapy crawl ai_job_spider
# 或启动增量采集
scrapy crawl ai_job_spider -a increment=true
```

## 5.3 生产环境部署流程

推荐使用 Docker Compose 进行生产环境部署，简化部署流程与环境配置。

### 步骤 1：配置环境变量

在项目根目录创建 `.env` 文件，配置生产环境变量：

```text

# MySQL 配置
MYSQL_HOST=mysql
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=xxx
MYSQL_DB=ai_workplace

# Redis 配置
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=xxx

# MinIO 配置
MINIO_ENDPOINT=minio:9000
MINIO_ACCESS_KEY=xxx
MINIO_SECRET_KEY=xxx
MINIO_BUCKET=ai-workplace-data
```

### 步骤 3：配置 Nginx 反向代理（可选）

为提升服务稳定性与安全性，可配置 Nginx 作为反向代理，转发前端与后端请求：

```nginx

server {
    listen 80;
    server_name your_domain.com;  # 替换为你的域名

    # 前端请求转发
    location / {
        proxy_pass http://frontend:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 后端 API 请求转发
    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源转发（MinIO）
    location /static {
        proxy_pass http://minio:9000/ai-workplace-data;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 5.4 常见部署问题排查

- 问题 1：后端服务启动失败，提示数据库连接超时？
解决方案：检查 MySQL 服务是否正常运行，`.env` 文件中 MySQL 配置是否正确，确保数据库端口可访问。

- 问题 2：前端页面无法加载，提示 API 请求失败？
解决方案：检查后端服务是否正常启动，前端配置文件 `frontend/.env.production` 中的 API 基础地址是否正确，确保前后端服务网络互通。

- 问题 3：数据采集失败，提示爬虫被拦截？
解决方案：检查爬虫配置中的请求头（User-Agent、Cookie 等）是否正确，可添加代理 IP 池规避反爬机制，或降低采集频率。

- 问题 4：MinIO 无法上传文件？
解决方案：检查 MinIO 服务是否正常运行，访问密钥与密钥是否正确，存储桶是否已创建且配置了正确的访问权限。
